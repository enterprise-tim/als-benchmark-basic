🎯 Purpose

This project helps answer the critical question: "What is the overhead of using AsyncLocalStorage, and is there anything I should be worried about?"

AsyncLocalStorage is a powerful Node.js feature that enables context propagation across asynchronous operations, but it comes with performance implications that vary significantly across Node.js versions and usage patterns.

🛠 Installation

1

Clone Repository

git clone https://github.com/tobrien/async-node-stats.git cd async-node-stats
2

Install Dependencies

npm install
3

Run Tests

npm start all

Requirements

  • Node.js: 16.0.0 or higher
  • NVM: For multi-version testing (optional)
  • Git: For version control and workflows

Quick Start

Run All Tests

npm start all

This will run both performance benchmarks and memory tests, then generate a comprehensive report.

Individual Test Categories

# Performance benchmarks only npm run benchmark # Memory profiling only npm run memory-test # Generate reports from existing results npm run generate-report

Multi-Version Testing

Test across multiple Node.js versions using NVM:

./test-versions.sh

This will automatically:

  1. Install and test Node.js versions 16.20.2, 18.19.1, 20.11.0, 21.7.3, 22.0.0, and 24.6.0
  2. Run benchmarks and memory tests for each version
  3. Generate a comprehensive comparison report

📈 Understanding the Results

Performance Metrics

  • Overhead Percentage: How much slower operations are with AsyncLocalStorage
  • Memory Impact: Additional memory usage in MB
  • Nested Overhead: Performance impact of nested AsyncLocalStorage contexts

✅ Interpretation Guidelines

  • < 5% overhead: Minimal impact, safe for most use cases
  • 5-15% overhead: Moderate impact, consider for high-throughput applications
  • > 15% overhead: Significant impact, use with caution
